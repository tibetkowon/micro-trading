# Study Mentor Skill (For Java Developer)

## Purpose
- 구현된 코드의 흐름과 파이썬/JS의 핵심 개념을 별도의 학습 문서로 작성하여 고원 님의 학습을 돕는다.
- 코드 내 주석은 최소화하고, 모든 상세 설명은 `docs/study/` 하위 문서에서 다룬다.

## Documentation Rules
- **Location**: 모든 문서는 `docs/study/{기능명}_study.md` 형식으로 저장한다.
- **Content Structure**:
    1. **Overview**: 이 기능이 왜 필요한가?
    2. **Logic Flow**: 데이터가 어떻게 흘러가는지 단계별로 설명 (Step-by-step).
    3. **Pythonic Point**: 이 코드에서 사용된 파이썬 특유의 기법이나 라이브러리 설명.
    4. **Java vs Python**: 자바(Spring)라면 어떻게 짰을지, 파이썬에서는 왜 이렇게 짰는지 비교.
    5. **Key Concept**: 초보자가 놓치기 쉬운 비동기(Async) 처리나 메모리 관리 포인트.

## Trigger
- 새로운 기능을 구현하거나 대규모 리팩토링을 마친 후, 반드시 "학습 문서를 생성할까요?"라고 먼저 제안한다.
